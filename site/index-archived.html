[ 03 ]

Nothing slips through.

A workflow without checkpoints forgets itself. This one records the decisions, the timing, and the context at every step — so you can always see what happened, and always pick up exactly where you left off.

You can't improve what you can't see — and invisible work becomes invisible failure. So every meaningful moment in the build loop leaves a written trace: the decisions, the timing, the context. Some of this the harness always did; the build record and the clean handoff note are the newest pieces. Together they're one thing — a way of working where nothing falls through the cracks.

DECISIONS

Every choice, with its reasons.

When a real decision gets made — a tool, a tradeoff, a direction — it's written down with the options weighed and why this one won. Months later you see not just what was chosen, but why.

RECOVERY

Stop anytime, lose nothing.

Close the laptop mid-task, reset, come back next week — work resumes already knowing what you were doing, what's next, and what was blocking you.

TRAIL

A dated record of every step.

Each step leaves a timestamped, written trace. Look back and see exactly what happened, in order — no reconstructing it from memory.

BUILD RECORD

Every round of work, written down.

When a piece of work finishes, the harness saves a short record of what got built, what was tested, and where it landed — so the busiest part of the loop is no longer invisible.

HANDOFF

A clean handoff note, every time.

Before you step away, the harness keeps a tidy, fixed-shape summary — current task, what's next, blockers — that the next session reads instantly.

[ 04 ]

Locked decisions.

Excerpts from PLAN.md — the choices that make this work, and the ones we deliberately said no to.

cat PLAN.md | grep -A1 'decisions locked' 1.2 KB
form factor
Claude Code / Codex plugin — slash commands, auto-skills, a single hook. No web app, no daemon.
memory model
Tiered markdown — hot memory.md auto-injected at SessionStart, cold archive/NNNN-*.md append-only. No vector DB.
verify gate
Agent walks every unchecked item in ACCEPTANCE.md against actual code, tests, and deploy. Evidence required.
plan gate
BUILD phase is entered only after explicit user approval. The plan IS the spec — no accidental coding.
tdd gate
BUILD refuses to edit implementation files until a declared failing test is observed in red. Trivial-task and explicit-bypass paths are logged to memory.md.
deployed verify
ACCEPTANCE rows tagged [deployed] are never evaluated locally. /bencium-deploy step 5.5 re-walks them against deploy.url. No smoke, no shipped.
opinionation
Templates ship neutral. Your opinions live in .harness/rules.md, so the harness adapts to your stack.
